# Astro Starter Template CLI

Цей пакет надає CLI `create-astro-starter` для швидкого створення багатомовних проєктів на Astro з готовими шаблонами та інтеграцією Tailwind і Zustand. Він поширюється як npm-пакет `create-astro-starter-template-znu`. 

## Встановлення

Використовуйте останню версію напряму через `npx` (рекомендується):

```bash
npx create-astro-starter-template-znu@latest
```

За потреби можна встановити CLI глобально:

```bash
npm install -g create-astro-starter-template-znu
create-astro-starter
```

## Створення нового проєкту

1. Запустіть CLI і вкажіть теку майбутнього проєкту:
   ```bash
   npx create-astro-starter-template-znu@latest my-app
   ```
2. Оберіть шаблон. Доступні варіанти `default` (з готовими сторінками, хедерами/футерами та i18n) або `empty` (мінімальна заготовка).
3. На кроці «Select languages to include» оберіть мови, які потрібні у вашому проєкті. За замовчуванням попередньо вибрані українська та англійська.
4. Після генерації перейдіть у створену теку, встановіть залежності та запустіть дев-сервер:
   ```bash
   cd my-app
   npm install
   npm run dev
   ```

## Що отримує проєкт

- **Скрипти розробки** — стандартні команди `npm run dev`, `npm run build`, `npm run preview` та `npm run astro` доступні у кожному шаблоні.
- **Багатомовність** — генерується утиліта `utils/i18n.ts` з вибраними мовами та типовою мовою за замовчуванням, а зайві JSON-файли локалізацій автоматично видаляються.
- **Стек** — шаблон `default` одразу містить Astro 5, Tailwind CSS 4 та Zustand для керування станом.

## Подальша розробка

Після запуску дев-сервера відкрийте `http://localhost:4321` і починайте працювати зі сторінками та компонентами, що створені у вибраному шаблоні. За потреби редагуйте переклади у каталозі `i18n` або додавайте нові мови, повторивши генерацію або оновивши `utils/i18n.ts` вручну.
